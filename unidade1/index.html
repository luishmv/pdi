1.1 Região Negativa em uma Imagem

Esse programa recebe uma imagem, processa ela em escalas de cinza e solicita ao usuário a coordenada de dois pontos: P1 e P2, dentro dos limites do tamanho da imagem. A região definida pelo retângulo de vértices opostos definidos por esses pontos será exibido com o negativo da imagem na região correspondente.

O código regions.cpp é descrito a seguir. Observa que o que é feito basicamente é percorrer os laços das coordenadas x e y delimitados pelos limites dos pontos impostos pelo usuário, subtraindo 255 dos valores dos pixels nesses pontos.

Código

Codigo 1 - regions.cpp

  #include <iostream>
  #include <opencv2/opencv.hpp>

  {
    Mat image;
    Point P1, P2;
  
    image = imread(argv[1], IMREAD_GRAYSCALE);
  
    if (!image.data)
    {
      cout << "Erro ao abrir a imagem" << endl;
      return 0;
    }
  
    cout << "Dimensões da imagem: " << image.rows << "*" << image.cols << endl;
  
    cout << "Digite as coordenadas do primeiro ponto: " << endl;
    cout << "x:";
    cin >> P1.x;
    cout << "y:";
    cin >> P1.y;
    cout << "Digite as coordenadas do segundo ponto: " << endl;
    cout << "x entre " << P1.x  << " e " << image.rows << ":";
    cin >> P2.x;
    cout << "y entre " << P1.y << " e " << image.cols << ":";
    cin >> P2.y;
  
    for (int i = P1.x; i <= P2.x; i++)
    {
      for (int j = P1.y; j <= P2.y; j++)
      {
        image.at<uchar>(i, j) = 255 - image.at<uchar>(i, j);
      }
    }
  
    imshow("Imagem com região negativada", image);
    imwrite("im_neg.png", image);
    waitKey();
  
    return 0;
  }

1.2. Troca de Quadrantes

Esse segundo exercício pede para implementar um programa que faça a troca de quadrantes da imagem, sendo o superior esquerdo trocado com o inferior direito e o superior direito trocado com o inferior esquerdo. Para tal, foi implementado um programa que primeiramente realizou a cópia da imagem em uma outra variável. Essa nova variável foi manipulada para que primeiramente houvesse a troca do primeiro e do quarto quadrante, e em seguinte foi feita a troca do segundo com o terceiro quadrante. O código trocaregioes.cpp pode ser conferido abaixo.

Codigo 2 - trocaregioes.cpp

    #include <iostream>
    #include <opencv2/opencv.hpp>
    
    using namespace cv;
    using namespace std;
    
    int main(int, char**){
      Mat image, imgModificada;
      int i, j;
      
      image = imread("cat.jpg", IMREAD_GRAYSCALE);
      
      if(!image.data) 
        cout << "nao abriu cat.jpg" << endl;
        
      int width = image.size().width;
      int height = image.size().height;
    
      namedWindow("imagem original", WINDOW_AUTOSIZE);
      namedWindow("imagem modificada", WINDOW_AUTOSIZE);
      
      image.copyTo(imgModificada);
      
      //Primeiro Quadrante - (y, x): (0, 0) -> (width/2, height/2)
      //Segundo Quadrante - (y, x): (width/2, 0) -> (width, height/2)
      //Terceiro Quadrante - (y, x): (0, height/2) -> (width/2, height)
      //Quarto Quadrante - (y, x): (width/2, height/2) -> (width, height)
      
      //for utilizado para trocar o primeiro e quarto quadrante
      for(i = 0; i < width/2; i++){
        for(j = 0; j < height/2; j++){
    	imgModificada.at<uchar>(i, j) = image.at<uchar>(i + width/2, j + height/2);
    	imgModificada.at<uchar>(i + width/2, j + height/2) = image.at<uchar>(i,j);
        }
      }
    
      //for utilizado para trocar o segundo e terceiro quadrante
      for(i = width/2; i < width; i++){
          for(j = 0; j < height/2; j++){
            imgModificada.at<uchar>(i, j) = image.at<uchar>(i - width/2, j + height/2);
            imgModificada.at<uchar>(i - width/2, j + height/2) = image.at<uchar>(i, j);
        }
      }
      
      imshow("imagem original ", image);
      imshow("imagem modificada", imgModificada); 
      imwrite("imgModificadaQ2.jpg", imgModificada);
       
      waitKey();
      return 0;
    }

2. Preenchendo Regiões

2.1. Contando Objetos em uma Cena

O programa inicialmente apresentado mostrava como rotular uma imagem binária, ou seja, que só possui dois valores de pixels possíveis, nesse caso 0 (puramente preto) ou 255 (puramente branco). Ao identificar um pixel de valor 255 (branco), é utilizado o algoritmo floodfill para descobrir os aglomerados de pixels, e agrupá-los em objetos.

Como o floodfill trabalha com uma contagem acumulatória e rotula os objetos com tons de cinza entre 0 e 255, quando há mais de 255 objetos na cena a rotulação fica comprometida, pois não há mais tons de cinza que possam ser utilizados para rotular o restante dos objetos.

Uma solução para esse problema seria resetar a contagem da variável nobjects, utilizando a função módulo. Dessa forma, quando o valor dela passa a ser maior ou igual a 255, a escala de cinza passa a ser zerada, possibilitando contar mais de 255 objetos. Essa solução proposta pode ser vista no código abaixo. 


    nobjects = 0;
    for (int i = 0; i < height; i++)
    {
        for (int j = 0; j < width; j++)
        {
            if (image.at<uchar>(i, j) == 255)
            {
                nobjects++;
                p.x = j;
                p.y = i;
                // utiliza a função módulo ao invés de um simples contador
                floodFill(image, p, nobjects % 255);
            }
        }
    }

Para criar um algoritmo aprimorado que identifica regiões com ou sem buracos internos, foi necessário seguir um conjunto de passos. O primeiro deles é eliminar as bolhas que tocam as bordas da imagem, pois não é possível dizer a priori se elas possuem buracos ou não.

Primeiro, são removidos os objetos da borda superior e da inferior: 


      //remove da borda superior e inferior
      for (int i = 0; i < width; i++)
      {
          if (image.at<uint8_t>(0, i) == 255)
              floodFill(image, Point(i, 0), 0);
          if (image.at<uint8_t>(height - 1, i) == 255)
              floodFill(image, Point(i, height - 1), 0);
      }

A seguida, são removidos os objetos das bordas laterais: 


    //remove das laterais
    for (int i = 0; i < height; i++)
    {
        //lateral esquerda
        if (image.at<uint8_t>(i, 0) == 255)
            floodFill(image, Point(0, i), 0);
        //lateral direita
        if (image.at<uint8_t>(i, width - 1) == 255)
            floodFill(image, Point(width - 1, i), 0);
    }

Depois, o background da imagem é preenchido com uma nova cor intermediária entre o preto e o branco de modo a diferenciá-lo dos buracos das bolhas, e tornando mais fácil de identificar esses buracos. As bolhas com buracos são identificadas, incrementando no contador de bolhas de buracos e “apagando” essas bolhas da imagem (preenchendo elas com a cor do background). 


    //troca o background, para facilitar a identificar os buracos das bolhas
    floodFill(image, cv::Point(0, 0), 127);
    imshow("Passo 2", image);
    imwrite("passo2.png", image);
    for (int i = 0; i < height; i++)
        for (int j = 0; j < width; j++)
        {
            //identifica uma bolha com buraco
            if (image.at<uint8_t>(i, j) == 0 && image.at<uint8_t>(i, j - 1) == 255)
            {
                //soma um no numero de bolhas e "apaga" a bolha encontrada
                nbolhas_com_buracos++;
                floodFill(image, Point(j - 1, i), 127);
            }
        }

Por último, são contadas as bolhas sem buracos. Primeiro essas bolhas que restaram são identificadas, e depois o contador dessas bolhas é incrementado e elas são “apagadas”, ou seja, preenchida com a cor do background. 


    //conta bolhas sem buracos
    for (int i = 0; i < height; i++)
        for (int j = 0; j < width; j++)
        {
            //identifica uma bolha
            if (image.at<uint8_t>(i, j) == 255)
            {
                //soma um no numero de bolhas sem buracos e "apaga" a bolha encontrada
                nbolhas_sem_buracos++;
                floodFill(image, Point(j, i), 127);
            }
        }

3.2. Motion Detector

Nesse exercício, é explorado o conceito de motion detector, ou detector de movimento. Utilizando os conceitos trabalhados de histogramas de imagens, é pedido um programa que seja capaz de capturar imagens em tempo real através de uma câmera disponibilizada pelo dispositivo que diga quando houver movimento.

Para fazer a implementação desse programa, teve-se de considerar uma função que identificasse o quanto um histograma de uma imagem difere do histograma da imagem anterior. Utilizando esse conceito, se o valor dessa função varia muito de uma imagem para outra significa que provavelmente houve movimento. Assim, se utilizou da função compareHist() do OpenCV para calcular a correlação entre dois histogramas, sendo um deles o da imagem capturada no instante atual e o outro da imagem capturada no instante antes. Quando esse valor de correlação é menor que 0.9, é considerado que houve movimento.

O código completo pode ser conferido aqui: motiondetector.cpp

4. Filtragem no domínio espacial I

4.1. Laplaciano do Gaussiano

Esse exercício explora o uso da filtragem em imagens em tons de cinza. O código filtroespacial.cpp, disponibilizado pelo professor, implementa diversos tipos de filtro, entre eles o filtro média, o filtro laplaciano e o filtro gaussiano, para serem aplicados em imagens capturadas em tempo real em tons de cinza, demonstrando o efeito de cada um desses filtros. A tarefa pede que seja adicionada mais uma opção de filtro para o programa, no caso o filtro laplaciano do gaussiano.

Para tal, deve ser feita uma modificação no código para que possa haver a aplicação de dois filtros simultaneamente em uma imagem. No caso do filtro laplaciano do gaussiano, deve ser primeiro aplicado o filtro gaussiano e depois o laplaciano. Assim, depois que a tecla correspondente ao laplaciano do gaussiano for pressionada e o filtro gaussiano for aplicado, deve-se aplicar o filtro laplaciano a essa imagem já filtrada para que os efeitos de ambos os filtros sejam mostrados ao usuário. Isso foi feita através de uma variável auxiliar booleano lg que diz se o filtro selecionado pelo usuário, correspondente a opção j, é o laplaciano do gaussiano ou não antes de exibir a imagem pro usuário. Caso seja, o filtro laplaciano é aplicado a imagem previamente filtrada. Esses trechos de edição podem ser vistos abaixo:

5. Filtragem no domínio espacial II

5.1. Tilt-Shift

Essa seção trabalha com o conceito de tilt-shift. O tilt-shift é um estilo de fotografia que se refere ao uso de picagem/inclinação para limitação ou aumento da profundidade de campo. Ela engloba dois tipos de movimento: a inclinação do plano óptico da objetiva em relação ao plano da filmagem (tilt) e o movimento paralelo da lente relativo ao plano da imagem (shift).

No primeiro exercício, pede-se que baseado no código disponibilizado pelo professor, o addweighted.cpp, se implemente um programa que possibilite que três ajustes sejam feitas a imagem na tela de interface: um ajuste para regular a altura da região central que entrará em foco; um ajuste para regular a força de decaimento da região borrada; e um ajuste para regular a posição vertical do centro da região que entrará em foco.

O código implementado pode ser conferido aqui: tiltshift.cpp
